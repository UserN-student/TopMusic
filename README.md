# ЧАСТЬ 1 — ДЛЯ ПОЛЬЗОВАТЕЛЯ

Полное описание каждой функции и каждого элемента интерфейса.

---

## Что такое TopMusic

TopMusic — это десктопный музыкальный плеер для Windows, построенный на Electron. Он работает полностью офлайн, хранит вашу библиотеку локально и не требует интернета для воспроизведения. Интерфейс тёмный, минималистичный, без лишних кнопок.

---

## Запуск приложения

```bash
npm start          # обычный запуск
npm start -- --dev # запуск с открытыми DevTools (для отладки)
```

---

## Интерфейс — общая структура

Окно делится на три зоны:

| Зона | Описание |
|------|----------|
| **Левая панель (сайдбар)** | Навигация, категории, кнопка добавления категории |
| **Центральная область** | Главная страница, плейлист, настройки |
| **Нижняя панель (плеер)** | Управление воспроизведением, прогресс, громкость |

Верхняя полоса окна — область перетаскивания (drag region). Кнопки управления окном (свернуть / развернуть / закрыть) находятся в правом верхнем углу.

---

## Сайдбар (левая панель)

### Логотип
Логотип TopMusic в верхней части сайдбара. Чисто декоративный элемент.

### Навигация
- **Главная** — открывает домашнюю страницу с карточкой текущего трека и блоком «Недавно прослушанные».
- **Настройки** — открывает страницу настроек.

### Категории
Под навигацией находится раздел «Категории»:
- **Все треки** — показывает полную библиотеку.
- **Понравившееся** — треки, которым вы поставили лайк.
- **Пользовательские категории** — создаются вручную (см. ниже). Каждая отображается со значком папки. Рядом с каждой пользовательской категорией при наведении появляется кнопка удаления (×).

### Кнопка «Добавить категорию»
Внизу сайдбара. Открывает модальное окно с полем ввода названия. После ввода названия и нажатия «Создать» категория появляется в сайдбаре и сохраняется в библиотеке.

---

## Главная страница

### Карточка «Сейчас играет»
Большая карточка в верхней части главной страницы. Показывает:
- **Обложку альбома** (если есть в метаданных файла). При наведении на обложку появляется кнопка play/pause.
- **Название трека** и **исполнителя**.
- **Дату добавления** трека в библиотеку.
- **Прогресс-бар** — кликабельный, показывает текущую позицию воспроизведения. Можно кликнуть для перемотки.
- **Время** — текущее и общее.
- **Кнопки управления**: перемешать, предыдущий, play/pause, следующий, повтор.
- **EQ-анимация** (четыре полоски) — анимируется когда трек играет, останавливается на паузе.

Если ничего не воспроизводится — карточка показывает «Ничего не воспроизводится».

### Блок «Недавно прослушанные»
Горизонтальный скролл-ряд карточек треков, которые вы слушали в этой сессии (до 8 штук). Карточки перемещаются — самый свежий трек всегда первый, остальные сдвигаются с анимацией.

**Взаимодействие с карточкой:**
- Клик по карточке другого трека — запускает его.
- Клик по карточке текущего трека — ставит на паузу / возобновляет.
- Правый клик — открывает контекстное меню трека.

**Скролл:** колесо мыши прокручивает ряд горизонтально. По краям ряда — плавное затухание (fade-эффект), чтобы карточки не обрезались резко.

**Кнопка «Очистить»** — очищает историю прослушивания.

## Страница «Все треки» / категории

### Верхняя панель
- **Поиск** — поле поиска по названию, исполнителю и формату файла. Поиск работает в реальном времени. Нажмите Escape чтобы сбросить.
- **Сортировка** — выпадающее меню с вариантами:
  - По названию: A→Z, Z→A
  - По дате добавления: сначала новые, сначала старые
  - По исполнителю: A→Z, Z→A
- **Кнопка «Настройки»** — быстрый переход в настройки.

### Карточки треков
Каждая карточка содержит:
- **Обложку** (или иконку ноты если обложки нет). При наведении появляется кнопка play/pause.
- **Название трека**.
- **Исполнителя** (или тип файла если исполнитель неизвестен).
- **Длительность**.
- **Значок сердца** если трек в избранном.
- **Кнопку «⋯»** — открывает контекстное меню.

**Клик по карточке:**
- Обычный клик — запускает трек.
- Клик по играющему треку — ставит на паузу / возобновляет.
- Ctrl+клик — выделяет трек (режим множественного выбора).
- Правый клик — контекстное меню.

**Кастомный цвет карточки** — через редактор трека можно задать цвет акцента. Карточка получит цветной фон и рамку.

### Пагинация
При количестве треков больше 60 внизу страницы появляются номера страниц. Клик по номеру переходит на страницу и прокручивает список наверх. Отображаются: первая страница, последняя, текущая и ±2 от текущей. Остальные заменяются «…».

### Панель выделения (множественный выбор)
При выделении одного или нескольких треков (Ctrl+клик или через контекстное меню) появляется панель:
- **Счётчик** выделенных треков.
- **«Выделить всё»** — выделяет все треки на текущей странице.
- **«В плейлист»** — добавляет выделенные треки в категорию.
- **«Удалить»** — удаляет выделенные треки (с подтверждением).
- **«Отмена»** — снимает выделение.

### Кнопки «Добавить трек» и «Добавить папку»
- **Добавить трек** — открывает диалог выбора файлов. Поддерживаемые форматы: MP3, WAV, OGG, FLAC, AAC, M4A, WMA, MP4.
- **Добавить папку** — открывает диалог выбора папки, добавляет все аудиофайлы из неё.

**Drag & Drop** — можно перетащить файлы прямо в окно приложения.

## Контекстное меню трека

Открывается правым кликом по карточке или кнопкой «⋯». Пункты:
- **Подробнее** — модальное окно с полной информацией о треке (название, формат, длительность, дата добавления, путь к файлу, путь к папке). Все поля можно скопировать кнопкой копирования.
- **Изменить** — открывает редактор трека (см. ниже).
- **Добавить в плейлист** — добавляет трек в одну или несколько категорий.
- **Выделить** — включает режим множественного выбора для этого трека.
- **Открыть в проводнике** — открывает папку с файлом в Windows Explorer с выделением файла.
- **Удалить** — удаляет трек из библиотеки (с подтверждением).

---

## Редактор трека

Открывается через контекстное меню → «Изменить». Позволяет:

- **Обложка** — загрузить своё изображение или GIF. Клик по превью открывает диалог выбора файла.
- **Название** — изменить отображаемое название трека.
- **Исполнитель** — изменить имя исполнителя.
- **Скорость воспроизведения** — слайдер от 0.25× до 3×. Применяется немедленно если трек играет.
- **Цвет карточки** — color picker для выбора цвета акцента карточки. Кнопка «Сброс» убирает цвет.
- **Эквалайзер** — 10-полосный EQ (32 Гц — 16 кГц), каждая полоса от −12 до +12 дБ.
- **Кнопка «Сбросить»** — возвращает все поля к исходным значениям.
- **Кнопка «Сохранить»** — сохраняет изменения.

Клик вне модального окна также сохраняет изменения.

## Нижний плеер

### Прогресс-бар
Тонкая полоска в самом верху нижней панели. Высота 4px, при наведении расширяется до 6px и появляется круглый ползунок.

**Перемотка:**
- Клик в любую точку — перемотка на эту позицию.
- Зажать и тянуть — плавная перемотка. Работает и когда трек на паузе.

### Информация о треке (левая часть)
- Обложка (42×42px).
- Название трека — если длинное, прокручивается как бегущая строка (можно отключить в настройках).
- Исполнитель — прокручивается в 2 раза медленнее названия.
- **Кнопка лайка** (сердечко) — добавляет/убирает трек из «Понравившегося».

### Элементы управления (центр)
- **Перемешать (shuffle)** — при активации треки воспроизводятся в случайном порядке. Кнопка подсвечивается белым когда активна.
- **Предыдущий трек** — переходит к предыдущему треку. Если прошло больше 5 секунд — перематывает в начало текущего.
- **Play/Pause** — большая круглая кнопка.
- **Следующий трек**.
- **Повтор** — три режима по кругу:
  - Выключен (серый).
  - Повтор всего списка (белый).
  - Повтор одного трека (белый + цифра «1» в углу кнопки).

### Время (правее центра)
Текущее время / общая длительность.

### Громкость (правая часть)
- **Иконка громкости** — клик переключает mute/unmute. Иконка меняется в зависимости от уровня (тихо / средне / громко / выключено).
- **Слайдер громкости** — клик устанавливает уровень.

## Настройки

### Статистика
Три карточки: общее количество треков, суммарное время всей библиотеки, количество треков в избранном.

### Аудио
- **Автовоспроизведение** — начинать воспроизведение при запуске (переключатель, сохраняется).
- **Продолжать с последнего трека** — запоминать позицию воспроизведения (переключатель).
- **Анимации интерфейса** — включить/выключить все CSS-анимации и переходы.
- **Бегущая строка** — включить/выключить прокрутку длинных названий в плеере. При выключении текст плавно возвращается на исходную позицию.

### Горячие клавиши
Справочная таблица всех горячих клавиш.

### О приложении
- Версия приложения.
- **Местоположение приложения** — кнопка «Открыть» открывает в проводнике папку с файлом данных `topmusic-data.json`.
- **Кнопка «Очистить библиотеку»** — удаляет все треки, историю и настройки (с подтверждением). Действие необратимо.

## Модальное окно загрузки

Появляется при добавлении файлов. Показывает:
- Спиннер загрузки.
- Заголовок текущей фазы.
- Прогресс-бар.
- Счётчик (обработано / всего).
- Текущий файл.
- **Смешную фразу** — меняется каждые 5 секунд, разная для каждой фазы.

**Фазы загрузки:**
1. **Загрузка файлов** — перебор и добавление файлов в список.
2. **Загрузка метаданных** — чтение тегов ID3, обложек, исполнителей.
3. **Определение длительности** — вычисление длины каждого трека.
4. **Отрисовка карточек** — рендеринг первой страницы плейлиста.
5. **Готово** — финальная фраза, модалка плавно исчезает через 1 секунду.

Фразы для каждой фазы хранятся в файле `loading-phrases.json` — их можно редактировать.

## Уведомления (тосты)

Небольшие всплывающие уведомления в правом нижнем углу. Появляются при:
- Добавлении треков.
- Сохранении изменений.
- Копировании в буфер обмена.
- Очистке истории.
- Включении/выключении настроек.

Три типа: успех (зелёный), ошибка (красный), информация (синий).

## Хранение данных

Все данные библиотеки сохраняются в файл `topmusic-data.json` в папке данных приложения (`%APPDATA%\topmusic\`). Файл создаётся автоматически. Открыть его расположение можно через Настройки → «Открыть».

**Что сохраняется:**
- Список всех треков с метаданными, обложками (base64), длительностями.
- Пользовательские категории и их содержимое.
- История недавно прослушанных.
- Настройки: громкость, mute, shuffle, режим повтора.

**Автосохранение** — данные сохраняются автоматически с задержкой 2 секунды после любого изменения (debounce).

MP3, WAV, OGG, FLAC, AAC, M4A, WMA, MP4

## Что происходит если файл не найден

При запуске приложение проверяет существование каждого файла в библиотеке. Если файл не найден — появляется модальное окно с именем и путём файла. Можно скопировать путь или удалить трек из библиотеки.

## `config.js`

Единственный файл, который используется и в main-процессе, и в renderer-процессе.

```js
const appConfig = {
  version: '1.2.8',          // версия приложения
  appName: 'TopMusic',
  minWindowWidth: 900,        // минимальная ширина окна
  minWindowHeight: 600,       // минимальная высота окна
  defaultVolume: 0.7,         // громкость по умолчанию (0–1)
  supportedFormats: [...]     // список расширений аудиофайлов
}
```

Экспортируется через `module.exports` для main.js и через `window.appConfig` для renderer.js.

## `main.js`

Главный процесс Electron. Запускается первым, создаёт окно браузера, регистрирует все IPC-обработчики.

### Константы и переменные

| Имя | Описание |
|-----|----------|
| `DEBUG` | Флаг отладки. При `true` выводит подробные логи в консоль |
| `DATA_PATH` | Путь к файлу данных: `%APPDATA%\topmusic\topmusic-data.json` |
| `mainWindow` | Ссылка на главное окно BrowserWindow |

### Кастомный протокол `localfile://`

Регистрируется до `app.ready` через `protocol.registerSchemesAsPrivileged`. Обрабатывает запросы вида `localfile:///C:/music/song.mp3`.

Поддерживает **Range-запросы** (заголовок `Range: bytes=X-Y`) — необходимо для перемотки аудио без полной загрузки файла. Определяет MIME-тип по расширению файла.

### `createWindow()`
Создаёт `BrowserWindow` с параметрами:
- `frame: false` — кастомный заголовок окна
- `backgroundColor: '#0a0a0a'` — цвет фона до загрузки
- `contextIsolation: true`, `nodeIntegration: false` — безопасность
- `sandbox: false` — нужно для preload-скрипта

### IPC-обработчики

#### Файловая система
| Канал | Описание |
|-------|----------|
| `open-files` | Диалог выбора аудиофайлов. Возвращает массив путей |
| `open-folder` | Диалог выбора папки. Возвращает путь |
| `get-audio-files-in-folder` | Читает папку, возвращает пути всех аудиофайлов |
| `check-file-exists` | Проверяет существование файла через `fs.access` |
| `open-folder-with-file` | Открывает папку в проводнике с выделением файла (`shell.showItemInFolder`) |
| `open-app-data-folder` | Открывает папку с `topmusic-data.json` в проводнике |
| `get-file-url` | Кодирует путь к файлу в URL формата `localfile:///...` |

#### Данные
| Канал | Описание |
|-------|----------|
| `save-data` | Сериализует данные в JSON и записывает в `DATA_PATH`. Поля начинающиеся с `_` не сохраняются. Обложки без префикса `data:` отбрасываются. `Set` конвертируется в `Array` |
| `load-data` | Читает `DATA_PATH`, парсит JSON. `trackPaths` конвертируется обратно в `Set` |
| `get-loading-phrases` | Читает `loading-phrases.json` из директории приложения |

#### Аудио
| Канал | Описание |
|-------|----------|
| `get-audio-duration` | Парсит заголовки файла без полного чтения. Поддерживает MP3 (ID3 + первый фрейм), WAV (RIFF-заголовок), FLAC (STREAMINFO-блок). Возвращает длительность в секундах или `null` |
| `get-audio-metadata` | Читает ID3-теги через библиотеку `jsmediatags`. Возвращает `{title, artist, album, year, genre, track, cover}`. Обложка кодируется в base64 data URL |

#### Управление окном
| Канал | Описание |
|-------|----------|
| `window-minimize` | Сворачивает окно |
| `window-maximize` | Разворачивает / восстанавливает окно |
| `window-close` | Закрывает окно |
| `window-is-maximized` | Возвращает `true/false` |

## `preload.js`

Выполняется в контексте renderer-процесса, но имеет доступ к Node.js API. Использует `contextBridge.exposeInMainWorld` для безопасного экспорта API в `window.electronAPI`.

Каждый метод — обёртка над `ipcRenderer.invoke()` или `ipcRenderer.send()`.

```js
window.electronAPI = {
  // Окно
  windowMinimize, windowMaximize, windowClose, windowIsMaximized,
  // Файлы
  openFiles, openFolder, getAudioFilesInFolder,
  checkFileExists, openFolderWithFile, openAppDataFolder, getFileUrl,
  // Данные
  saveData, loadData, getAudioMetadata, getAudioDuration,
  // Фразы
  getLoadingPhrases
}
```

## `renderer.js`

Самый большой файл (~2700 строк). Весь UI-код. Работает в контексте Chromium, не имеет прямого доступа к Node.js.

### Глобальные переменные состояния

| Переменная | Тип | Описание |
|-----------|-----|----------|
| `tracks` | `Array` | Массив всех треков библиотеки |
| `playlists` | `Object` | Словарь категорий `{id: {name, trackPaths: Set}}` |
| `currentIndex` | `number` | Индекс текущего трека в `tracks[]`, -1 если ничего не играет |
| `isPlaying` | `boolean` | Состояние воспроизведения |
| `isShuffle` | `boolean` | Режим перемешивания |
| `repeatMode` | `0\|1\|2` | 0=выкл, 1=повтор списка, 2=повтор трека |
| `isMuted` | `boolean` | Состояние mute |
| `lastVolume` | `number` | Последний ненулевой уровень громкости |
| `currentPage` | `string` | Текущая страница навигации |
| `currentCategory` | `string` | ID текущей категории |
| `customCategories` | `Array` | Пользовательские категории `{id, name}` |
| `recentlyPlayed` | `Array` | Пути треков в порядке прослушивания (до 20) |
| `selectedTracks` | `Set` | Пути выделенных треков |
| `searchQuery` | `string` | Текущий поисковый запрос |
| `_currentDisplayTracks` | `Array` | Кэш отсортированного списка для пагинации |
| `_currentPage` | `number` | Текущая страница пагинации (0-based) |
| `marqueeEnabled` | `boolean` | Включена ли бегущая строка |

### Структура объекта трека

```js
{
  path: string,          // абсолютный путь к файлу
  name: string,          // название (из тега или имени файла)
  ext: string,           // расширение в верхнем регистре ('MP3', 'FLAC', ...)
  duration: number|null, // длительность в секундах
  dateAdded: number,     // timestamp добавления
  liked: boolean,        // в избранном
  categories: string[],  // массив ID категорий
  artist?: string,       // исполнитель (из тега)
  album?: string,        // альбом (из тега)
  cover?: string,        // обложка в формате data:image/...;base64,...
  cardColor?: string,    // кастомный цвет карточки (#rrggbb)
  speed?: number,        // скорость воспроизведения (0.25–3)
  eq?: number[],         // 10 значений эквалайзера (-12..+12)
  _metaLoaded?: boolean  // служебный флаг (не сохраняется в JSON)
}
```

### Инициализация — `init()`

Вызывается при загрузке страницы. Последовательность:
1. Устанавливает громкость из `lastVolume`
2. `loadData()` — загружает данные из файла
3. Рендерит сайдбар, недавние, синхронизацию
4. `setupEventListeners()` — навешивает все обработчики событий
5. `checkMissingFiles()` — проверяет существование файлов
6. `loadTrackDurations()` / `loadAllMetadata()` — фоновая загрузка для старых треков

### Добавление файлов

**`addFiles()`** и **`addFolder()`** — асинхронные функции с одинаковой логикой:
1. Открывают диалог выбора
2. Показывают `showLoadingModal()`
3. Перебирают файлы, добавляют новые в `tracks[]`
4. Вызывают `loadMetadataInModal()` — загрузка метаданных + рендеринг внутри модалки
5. `hideLoadingModal()` — плавное закрытие
6. `loadTrack(0)` если библиотека была пустой

**`handleDrop(e)`** — обработчик drag & drop, та же логика.

### Загрузка метаданных — `loadMetadataInModal()`

Три последовательные фазы, все выполняются пока модалка открыта:

**Фаза 1 — metadata:** 4 параллельных воркера читают теги через `getAudioMetadata`. Обновляют `track.name`, `track.artist`, `track.album`, `track.cover`.

**Фаза 2 — processing (длительности):** 4 параллельных воркера. Сначала пробует `getAudioDuration` (парсинг заголовков), при неудаче — `getDurationViaAudio` (Audio element с таймаутом 8с).

**Фаза 3 — processing (рендеринг):** Вызывает `renderPlaylistInBackground()` — рендерит первую страницу батчами по 15 карточек через `setTimeout(0)` между батчами.

### Рендеринг плейлиста — `renderPlaylist(page?)`

1. Фильтрует `tracks[]` по текущей категории и поисковому запросу
2. Сортирует по `currentSortType`
3. Сохраняет в `_currentDisplayTracks`
4. Вычисляет страницу: `startIdx = _currentPage * PAGE_SIZE` (PAGE_SIZE = 60)
5. Рендерит карточки батчами по 20 через `requestAnimationFrame`
6. Добавляет пагинацию если страниц > 1
7. Вызывает `loadMetadataForPageIfNeeded()` для треков без метаданных

### Построение карточки — `buildTrackCard(track, realIndex, isSelecting)`

Создаёт DOM-элемент карточки. Если `track.cover` есть — вставляет `<img>` с `loading="lazy"`. Навешивает обработчики: клик (play/pause/выделение), контекстное меню, кнопка «⋯».

### Пагинация — `buildPaginationHTML(totalPages, currentPage)`

Генерирует HTML кнопок страниц. Алгоритм: всегда показывает первую, последнюю, текущую и ±2 от текущей. Промежутки заменяются «…». `window.goToPage(page)` — переход на страницу + скролл наверх + загрузка длительностей для новой страницы.

### Плеер

**`loadTrack(index)`** — загружает трек:
- Устанавливает `audio.src` через `buildFileUrl()`
- Обновляет UI (название, исполнитель, обложка, лайк)
- Добавляет в `recentlyPlayed` → `renderRecentAnimated()`
- Применяет скорость воспроизведения
- Запускает `startMarquee()`

**`buildFileUrl(filePath)`** — конвертирует Windows-путь в `localfile:///C:/music/song.mp3`. Кодирует каждый сегмент пути через `encodeURIComponent`, сохраняя букву диска.

**`play()` / `pause()`** — управляют `audio.play()/pause()`, обновляют иконки, запускают/останавливают `progressInterval`.

**`startProgress()`** — `setInterval` каждые 100мс. Обновляет `progressFill.style.width` и время. Также обновляет прогресс-бар на карточке «Сейчас играет».

**`setupDraggableProgress()`** — обработчики `mousedown/mousemove/mouseup` и `touchstart/touchmove/touchend` на прогресс-баре. При начале перетаскивания запоминает `wasPlayingBeforeDrag`, паузирует аудио. При отпускании — возобновляет если играло.

**`handleNext()`** — логика следующего трека:
- `repeatMode === 2`: перемотка в начало текущего
- `isShuffle`: случайный индекс (не равный текущему)
- Иначе: `(currentIndex + 1) % tracks.length`

**`handleTrackEnded()`** — вызывается по событию `audio.onended`:
- `repeatMode === 2`: перемотка в начало
- `repeatMode === 1`: `handleNext()`
- `isShuffle`: `handleNext()`
- Иначе: следующий по порядку или стоп в конце списка

### Бегущая строка — `startMarquee()`

Запускается при каждом `loadTrack()`. Проверяет переполнение текста в `.player-track-details`. Если текст шире контейнера:
- Добавляет класс `marquee-active` (показывает fade по краям)
- Цикл: стоит 1с → едет влево за `overflow/40` секунд → пауза 0.5с → возвращается → повтор
- Название движется со скоростью 40px/с, исполнитель — 20px/с (в 2 раза медленнее)

`stopMarquee()` — очищает все таймеры, плавно возвращает текст на место (`transition: 0.4s`).

### Недавно прослушанные

**`renderRecent()`** — рендерит до 8 карточек из `recentlyPlayed[]`. Клик по текущему треку — `togglePlay()`, по другому — `loadTrack()` + `play()`.

**`addToRecentlyPlayed(path)`** — добавляет путь в начало `recentlyPlayed[]`, удаляет дубликаты, обрезает до 20. Если трек уже был первым — не перерисовывает. Иначе вызывает `renderRecentAnimated()`.

**`renderRecentAnimated()`** — FLIP-анимация: запоминает `getBoundingClientRect()` всех карточек до перерисовки, рендерит заново, вычисляет дельту позиций и применяет `transform: translate(dx, dy)` с последующим сбросом через `requestAnimationFrame`. Новые карточки появляются с fade+scale.

### Загрузка метаданных (фоновая) — `_metaWorker()`

Используется для треков из старых сохранений (без `_metaLoaded`). Очередь `_metaQueue`, конкурентность `META_CONCURRENCY = 2`. Между треками пауза 30мс. Обновляет только player-bar и карточки в «Недавних» — карточки плейлиста обновятся при следующем рендере страницы.

### Загрузка длительностей — `loadTrackDurationsForPage()`

Грузит длительности только для треков текущей страницы (не всей библиотеки). Конкурентность 3. Между треками пауза 20мс. Обновляет длительность точечно в уже отрисованных карточках (без полного ре-рендера).

### Данные — `saveData()` / `loadData()`

**`saveData()`** — вызывает `window.electronAPI.saveData(data)`. Данные: `tracks`, `playlists`, `syncConfigs`, `customCategories`, `categoryCounter`, `recentlyPlayed`, `settings`.

**`saveDataDebounced()`** — обёртка с задержкой 2000мс. Предотвращает частые записи на диск.

**`loadData()`** — загружает данные, восстанавливает `Set` для `trackPaths`, сбрасывает `_metaLoaded` (служебный флаг), валидирует обложки (должны начинаться с `data:`), помечает треки с обложкой И исполнителем как `_metaLoaded = true`.

### Модальное окно загрузки

**`showLoadingModal(total)`** — async. Сначала вызывает `ensureLoadingPhrases()` (загружает JSON один раз), затем показывает модалку и запускает ротацию фраз фазы `loading`.

**`setLoadingPhase(phase, titleText)`** — переключает фазу: меняет заголовок, сбрасывает прогресс-бар, запускает ротацию фраз для новой фазы.

**`_startPhraseRotation(phase)`** — показывает случайную фразу сразу, затем `setInterval(5000)` для смены. При смене: fade out (0.5с) → новая фраза → fade in.

**`hideLoadingModal()`** — async. Показывает финальную фразу из `done`, ждёт 1000мс, затем `opacity: 0` за 1000мс, убирает класс `open`.

### Навигация — `window.switchPage(el)`

Переключает активный раздел. Управляет видимостью `homeWrapper`, `playlistView`, `settingsView`. Сбрасывает поиск и выделение при переходе. Сбрасывает `_currentPage = 0` при переходе в категорию.

### Редактор трека — `openEditTrackModal(trackIndex)` / `saveEditTrack(trackIndex)`

Заполняет поля модалки данными трека. Обложка загружается через `FileReader.readAsDataURL`. Сохраняет в `track._pendingCover` и `track._pendingColor` до нажатия «Сохранить». При сохранении применяет скорость к `audio.playbackRate` если это текущий трек.

## `loading-phrases.json`

Четыре массива строк для разных фаз загрузки:

| Ключ | Фаза | Когда показывается |
|------|------|--------------------|
| `loading` | Добавление файлов | Перебор файлов, добавление в список |
| `metadata` | Загрузка тегов | Чтение ID3, обложек, исполнителей |
| `processing` | Обработка | Длительности + рендеринг карточек |
| `done` | Завершение | Финальная фраза перед закрытием |

Фраза выбирается случайно из массива. Меняется каждые 5 секунд. Можно добавлять любое количество строк в каждый массив — формат не ограничен.


```
        ↓
addFiles() / addFolder() / handleDrop()
        ↓
```

## Безопасность

- `contextIsolation: true` — renderer не имеет доступа к Node.js API напрямую
- `nodeIntegration: false` — Node.js отключён в renderer
- `webSecurity: true` — стандартная политика безопасности Chromium
- Весь доступ к файловой системе — только через IPC-каналы в main.js
- Протокол `localfile://` зарегистрирован как `secure` и `bypassCSP` для корректного воспроизведения локальных файлов

## Известные ограничения

- WMA-файлы могут не воспроизводиться в зависимости от кодеков системы
- MP4 поддерживается только как аудио (видео не отображается, обложка не извлекается)
- Эквалайзер сохраняется в данных трека, но не применяется к Web Audio API (визуальный)
- Автовоспроизведение и «продолжать с последнего трека» — переключатели сохраняются, но логика не реализована (заглушки)
